# Architecture Decision Log

All major technical decisions documented here for future course-correction.

---

## ADR-001: Web Search Removal
**Date:** 2026-03-08
**Status:** Implemented
**Context:** Claude's web_search tool costs $0.67/session and inflates input tokens from ~5K to ~125K. With Sonnet 4.5, each article cost ~$1.10. At 500 articles/year = $550/year just for AI.
**Decision:** Remove web search entirely. Pre-fetch market data from external APIs and inject into prompt.
**Consequence:** Input tokens drop to ~15K. Need a reliable free data source.

---

## ADR-002: Model Switch — Sonnet to Haiku
**Date:** 2026-03-08
**Status:** Implemented
**Context:** Without web search, prompts are much simpler (~15K tokens). Sonnet is overkill. Haiku 4.5 is 3.75x cheaper on input, 3.75x cheaper on output.
**Decision:** Switch to claude-haiku-4-5-20251001 for all generation.
**Risk:** Haiku had 30% JSON failure rate WITH web search, but that was due to web search complexity. With structured pre-fetched data, JSON reliability should be fine.
**Fallback:** If Haiku quality drops, switch back to Sonnet. Cost difference at ~15K tokens is minimal ($0.02 vs $0.06/article).

## ADR-003: FMP Free Tier as Primary Data Source
**Date:** 2026-03-08
**Status:** Implemented
**Context:** Evaluated 5+ data APIs:
- **FMP free:** 250 calls/day, ~87 US tickers, reliable official API. No TSX.
- **Yahoo Finance (yahoo-finance2):** Free, unlimited, global coverage. BUT unofficial, unstable, breaks when Yahoo changes endpoints. Bad reviews on reliability.
- **Alpha Vantage:** 25 calls/day (too few), US-focused.
- **Twelve Data:** 800 calls/day, good coverage but API key required, mixed reliability.
- **Massive (Polygon):** Free tier = 5 calls/min, US only, EOD only.
- **IBKR:** Requires TWS/Gateway running 24/7, socket-based, overkill for daily data.

**Decision:** FMP free for US tickers it covers (~87) + market movers (gainers/losers/actives). Local stock JSON for everything else (qualitative context only).
**Why not Yahoo:** User researched reviews — too unstable for production. We tested it; v3 API requires constructor pattern, dailyGainers deprecated, endpoints change frequently.
**Why not IBKR:** User has account, but IBKR requires running TWS/Gateway on a machine 24/7. Not worth the infrastructure overhead for daily data fetches.

## ADR-004: Data Confidence Tagging
**Date:** 2026-03-08
**Status:** Implemented
**Context:** Local stock data (data/stocks/*.json) was generated by GPT-4o-mini on 2026-03-08. Contains approximate metrics like "P/E ~12-14", "dividend yield ~3.5%". These are guesses, not facts. Our RY roast got a D+ partly because of wrong dividend yield data.
**Decision:** Tag every data point with a confidence level:
- `VERIFIED` — from FMP API, real-time, cite exact numbers
- `APPROXIMATE` — from local JSON, qualitative only, say "approximately" or "historically around"
- `UNAVAILABLE` — no data at all, say "data not available"

Prompt guardrails enforce: NEVER fabricate specific numbers. If data is approximate, hedge the language.
**Consequence:** Articles for TSX stocks will be more qualitative (thesis-focused) than quantitative (number-focused). This is actually a stronger editorial angle — analyzing the *logic* of a recommendation rather than quoting specific ratios.

## ADR-005: Cost Tracking System
**Date:** 2026-03-08
**Status:** Implemented
**Context:** Need visibility into AI generation costs to monitor burn rate and prove the cost optimization is working.
**Decision:** Log every generation run to data/costs.json with: date, type, model, input/output tokens, USD cost, API calls, duration. Admin dashboard shows totals, monthly breakdown, per-run detail, and yearly projection.
**Files:** src/lib/costs.ts, src/app/api/admin/costs/route.ts, admin page costs tab.

## ADR-006: TSX Coverage Strategy
**Date:** 2026-03-08
**Status:** Accepted (trade-off)
**Context:** FMP free tier doesn't cover TSX tickers (RY, SHOP, ENB, etc). We have 37 TSX tickers. Options:
1. Pay $19/month for FMP Starter (full coverage) — $228/year
2. Use Yahoo Finance — unreliable
3. Use local data (qualitative) — no real numbers
4. Use IBKR — too complex

**Decision:** Use local data for TSX tickers, stripped of specific numbers. Focus analysis on recommendation logic, sector dynamics, competitive landscape. Haiku uses its training knowledge for general market context.
**Trade-off:** TSX articles won't quote exact current P/E, yield, or price. They'll focus on whether the publication's thesis makes sense.
**Future:** When revenue justifies it, upgrade to FMP Starter ($19/mo) for full TSX coverage. Or explore TMX direct data feeds.

## Open Questions / Future Decisions

1. **Batch API:** Should we switch to async batch generation (50% cost reduction, 24hr turnaround)? Makes sense for scheduled daily generation but breaks real-time /api/generate.
2. **TSX data source:** Revisit when revenue > $20/month. FMP Starter ($19/mo) gives full TSX + global coverage.
3. **Haiku reliability:** Monitor JSON parse failure rate. If >10%, consider switching back to Sonnet (cost diff is minimal at ~15K tokens).
4. **Local data refresh:** Current data/stocks/*.json is from 2026-03-08. Consider a monthly refresh script using FMP or a manual update process.
